# Моя стажировка

MVP: stazhirovka.rest

Отчёт в Google Documents: https://clck.ru/3TxSQU

Видео с демонстрацией работы: https://clck.ru/3TxSUE

«Моя стажировка» — веб-приложение для студентов, которое помогает искать стажировки и не теряться в процессе отбора. Сервис объединяет каталог стажировок, информацию о компаниях, отзывы, личную доску откликов и напоминания о следующих шагах. Вместо разрозненных ссылок, заметок и таблиц студент получает один инструмент для всего пути: от выбора подходящей стажировки до результата по отклику.

## Возможности

- Каталог стажировок с фильтрами по городу, формату работы, направлению, оплате и срокам подачи.
- Страницы компаний с описанием, ссылками, оформлением и отзывами.
- Личный трекер откликов в формате канбан-доски.
- Этапы подачи: хочу податься, подал, тестовое, интервью, оффер, отказ и архив.
- Задачи и даты следующего шага для каждой подачи.
- Уведомления о ближайших и просроченных действиях.
- Рейтинги компаний на основе пользовательских отзывов.
- Административная панель для управления стажировками, компаниями, направлениями и аккаунтами.

## Архитектура

Проект реализован как C++17 backend с локальным HTTP-сервером на POSIX sockets и базой данных SQLite.

Основные части приложения:

- `Application` — точка входа, загрузка настроек, подготовка базы данных и запуск приложения.
- `HttpServer` — сетевой слой, который принимает HTTP-запросы от браузера.
- `Router` — маршрутизатор, который сопоставляет URL и HTTP-метод с нужным обработчиком.
- `PublicController` — публичные страницы: главная, каталог, компании, уведомления и рейтинги.
- `ApplicationController` — личный трекер откликов пользователя.
- `AdminController` — административная часть для управления данными.
- `Database` — единый слой работы с SQLite.
- `PageRenderer` — формирование HTML-страниц и раздача статических файлов.
- Репозитории — классы доступа к данным для компаний, стажировок, откликов, пользователей и отзывов.

## Структура проекта

```text
cpp_backend/
  include/          Заголовочные файлы и интерфейсы классов
  src/              Реализация серверной логики
  static/           CSS и статические ресурсы
  Makefile          Сборка через make
  CMakeLists.txt    Сборка через CMake
```

## Требования

- C++17-совместимый компилятор
- SQLite3
- POSIX-совместимая среда для socket API
- make или CMake

## Запуск через make

```bash
cd cpp_backend
make
PORT=5050 ./internstart_cpp
```

После запуска приложение будет доступно по адресу:

```text
http://127.0.0.1:5050
```

Административная панель:

```text
http://127.0.0.1:5050/admin/login
```

## Запуск через CMake

```bash
cmake -S cpp_backend -B build
cmake --build build
PORT=5050 ./build/internstart_cpp
```

## Переменные окружения

Приложение читает `.env` и `.env.local` из корня проекта.

```env
HOST=127.0.0.1
PORT=5050
DATABASE_URL=sqlite:///internships.db
ADMIN_USERNAME=admin
ADMIN_PASSWORD=change_me
```

Если переменные не заданы, используются значения по умолчанию.

## Основные разделы сайта

- `/` — главная страница.
- `/internships` — каталог стажировок.
- `/companies` — список компаний.
- `/applications` — личная доска откликов.
- `/notifications` — уведомления и задачи.
- `/ratings` — отзывы и рейтинги компаний.
- `/admin/login` — вход в административную панель.

## База данных

При первом запуске приложение инициализирует SQLite-схему автоматически. В базе хранятся пользователи, компании, стажировки, направления, отклики, история этапов и отзывы. Для демонстрации предусмотрены стартовые данные, чтобы сайт можно было открыть и сразу проверить основные сценарии.
